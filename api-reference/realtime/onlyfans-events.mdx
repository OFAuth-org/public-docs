OnlyFans Events is a beta Realtime surface that relays supported upstream OnlyFans websocket traffic to your Svix-managed webhook endpoint.

<Info>
  **Beta status:** The top-level webhook contract is now documented as a stable OpenAPI 3.1 webhook, but the nested upstream payload families may continue to expand as OFAuth classifies more websocket activity.
</Info>

## What Is Stable

- The top-level event type is always `of.websocket.event.received`
- The webhook is always delivered over HTTPS through Svix
- `data.payloadKey` is the stable shape key you should use for filtering and routing
- `data.shapeSubtype` is the curated subtype label when OFAuth has promoted that payload family

## How To Integrate

1. Verify the Svix signature on the raw request body.
2. Route incoming deliveries by `type`.
3. For `of.websocket.event.received`, branch on `data.payloadKey`.
4. Parse `data.json` defensively and tolerate new payload families appearing over time.

## Notes

- `data.rawText` and `data.rawBase64` are preserved for beta debugging and fallback parsing.
- `data.truncated` tells you whether OFAuth had to truncate the upstream payload before delivery.
- A single `payloadKey` can represent many individual events over time. The shape catalog is intentionally keyed by payload family, not by request-specific values like message text or timestamps.
