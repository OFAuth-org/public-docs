## TypeScript / JavaScript

**Recommended: Use the SDK**

The OFAuth SDK provides the fastest path to production with full TypeScript support, built-in error handling, and automatic retry logic.

```typescript
import OFSDK from "@ofauth/onlyfans-sdk"

const sdk = new OFSDK({
  mode: "access",
  ofauthApiKey: process.env.OFAUTH_API_KEY
})

// Type-safe, auto-retry, structured errors
const { data, error } = await sdk.messages.sendChatMessage({
  authentication: { connectionId: "conn_123" },
  userId: 123,
  params: { text: "Hello!" }
})
```

## Other Languages

**Recommended: Use the REST API directly**

OFAuth's REST APIs work with any language. Make HTTP requests to our endpoints using your preferred HTTP client.

<CodeGroup>

```python Python
import requests

# Create a Link session
response = requests.post(
    "https://api.ofauth.com/v2/link/init",
    headers={
        "apikey": "YOUR_API_KEY",
        "Content-Type": "application/json"
    },
    json={
        "successUrl": "https://yourapp.com/success",
        "returnUrl": "https://yourapp.com/cancel"
    }
)

auth_url = response.json()["url"]
```

```go Go
package main

import (
    "bytes"
    "encoding/json"
    "net/http"
)

func createLinkSession(apiKey string) (string, error) {
    payload := map[string]string{
        "successUrl": "https://yourapp.com/success",
        "returnUrl":  "https://yourapp.com/cancel",
    }
    body, _ := json.Marshal(payload)

    req, _ := http.NewRequest("POST", "https://api.ofauth.com/v2/link/init", bytes.NewBuffer(body))
    req.Header.Set("apikey", apiKey)
    req.Header.Set("Content-Type", "application/json")

    client := &http.Client{}
    resp, err := client.Do(req)
    // Handle response...
}
```

```php PHP
$response = file_get_contents("https://api.ofauth.com/v2/link/init", false, stream_context_create([
    'http' => [
        'method' => 'POST',
        'header' => [
            'apikey: YOUR_API_KEY',
            'Content-Type: application/json'
        ],
        'content' => json_encode([
            'successUrl' => 'https://yourapp.com/success',
            'returnUrl' => 'https://yourapp.com/cancel'
        ])
    ]
]));

$authUrl = json_decode($response, true)['url'];
```

</CodeGroup>

## Direct Session Headers

**For: Managing your own OnlyFans sessions**

If you obtain OnlyFans session credentials through your own means, you can use them directly with the Access API. OFAuth will handle request signing while you manage the sessions.

```http
POST /v2/access/users/me HTTP/1.1
Host: api.ofauth.com
apikey: YOUR_API_KEY
x-of-cookie: YOUR_SESSION_COOKIE
x-of-user-agent: YOUR_USER_AGENT
x-of-bc: YOUR_BC_HEADER
x-of-user-id: YOUR_USER_ID
```

<Warning>
	**More responsibility, more complexity.** You're responsible for session expiry, rotation, and security. We recommend using Link + Connection IDs for most use cases. Direct session access is available through [Enterprise Whitelabel](/enterprise/whitelabel) for approved partners.
</Warning>

**Required headers:**
- `x-of-cookie` - Session cookie containing `fp`, `auth_id`, and `sess`
- `x-of-user-agent` - User agent string

**Optional headers:**
- `x-of-bc` - BC header value
- `x-of-user-id` - OnlyFans user ID

## Dynamic Rules Only

**For: Building your own OnlyFans proxy**

If you're making direct requests to OnlyFans APIs (not through OFAuth's Access API), you need current signing rules. OnlyFans changes these frequently - sometimes multiple times per day.

```javascript
// Get current signing rules
const rules = await fetch("https://api.ofauth.com/v2/dynamic-rules", {
  headers: { apikey: "YOUR_API_KEY" }
})

// Or let OFAuth sign a specific request
const signed = await fetch("https://api.ofauth.com/v2/dynamic-rules/sign", {
  method: "POST",
  headers: { 
    apikey: "YOUR_API_KEY",
    "Content-Type": "application/json"
  },
  body: JSON.stringify({
    endpoint: "/api2/v2/users/me",
    "user-id": "12345"
  })
})
```

<Warning>
	**Advanced use case.** Most integrations should use the Access API, which handles signing automatically. Dynamic Rules are for custom proxy implementations or edge cases.
</Warning>

| Feature | SDK | REST API | Session Headers | Dynamic Rules |
|---------|-----|----------|-----------------|---------------|

## Still Not Sure?

<AccordionGroup>

<Accordion title="I'm building a web app with Next.js / Express">
Use the **SDK**. It's designed for Node.js environments and provides the best developer experience with TypeScript.
</Accordion>

<Accordion title="I'm building a Python/Django/FastAPI backend">
Use the **REST API** directly. Make HTTP requests from your Python code to Link and Access endpoints.
</Accordion>

<Accordion title="I already have OnlyFans session cookies">
Use **Session Headers** with the Access API. You provide the session, OFAuth handles signing and proxying.
</Accordion>

<Accordion title="I'm building a complete OnlyFans client from scratch">
Use **Dynamic Rules** to get signing parameters, then make requests directly to OnlyFans. This is the most complex path.
</Accordion>

<Accordion title="I just want to prototype quickly">
Start with the **SDK** if you can use TypeScript, or the **REST API** with your favorite language. Both support our [Sandbox environment](/getting-started/environments/sandbox) for testing.
</Accordion>

</AccordionGroup>
