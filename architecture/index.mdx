## Overview

The OFAuth Public API v2 is built on modern REST principles with comprehensive
OpenAPI documentation. It provides a robust, enterprise-grade platform designed
to handle the complexities of OnlyFans integration while maintaining security,
performance, and reliability.

<Info>
	**New to OFAuth?** Start with our [Quick Start Guide](/quickstart) to get up
	and running in minutes, then return here for deeper architectural
	understanding.
</Info>

## API Structure

The OFAuth Public API v2 is organized into four main categories:

<CardGroup cols={2}>
	<Card title="Link" icon="link" href="/guides/link/overview">
		**Authentication & Connection Management** - Handle user authentication with
		hosted or programmatic flows
	</Card>
	<Card title="Access" icon="key" href="/api-reference/access/overview">
		**Proxied Data Access** - Seamless OnlyFans API access with automatic
		session management, supporting both Link connection IDs and direct session
		headers
	</Card>
	<Card title="Account" icon="user" href="/api-reference/account/overview">
		**Organization Management** - Webhook configuration, connection monitoring,
		and account settings
	</Card>
	<Card
		title="Dynamic Rules"
		icon="cog"
		href="/api-reference/dynamic-rules/overview"
	>
		**Request Signing** - Real-time rule updates and custom request signing for
		OnlyFans API changes
	</Card>
</CardGroup>

## Migration & Versioning

### API Versioning

- **URL Versioning**: `/v2/` in the path for major versions
- **Backward Compatibility**: Maintained for minimum 12 months
- **Deprecation Notices**: 90-day advance notice for breaking changes
- **Migration Guides**: Detailed guides for version upgrades

### Version Lifecycle

1. **Beta**: New features in beta for testing
2. **Stable**: Generally available
3. **Deprecated**: Marked for removal with timeline
4. **Sunset**: Version removed after migration period
