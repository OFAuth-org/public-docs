## Core Architecture

### SDK Instance

The SDK is initialized as a single instance that provides access to all modules:

```typescript

// Single SDK instance with global configuration

  mode: "access",           // ‚úÖ Validated mode options
  ofauthApiKey: "your-key", // ‚úÖ Required for managed access
  debugLog: false           // ‚úÖ Optional debugging
})

// Access modules through the instance
sdk.messages    // üí¨ Messaging operations
sdk.posts       // üìù Content management
sdk.user        // üë§ User operations
sdk.vault       // üóÉÔ∏è Media management
// ... all other modules
```

### Module Structure

Each module is **self-contained** and focused on a specific domain:

```typescript

  // All methods follow the same pattern
  async operation({
    authentication: AuthOptions,    // ‚úÖ Required auth context

```

## Type Safety System

### Automatic Parameter Validation

The SDK validates **all parameters** at compile time and runtime:

```typescript
// ‚úÖ TypeScript catches invalid parameters before runtime
const result = await sdk.statistics.getStats({
  authentication: { connectionId: "conn_123" },
  statType: "posts",              // ‚úÖ Validates against allowed enum values
  params: {
    by: "purchases",              // ‚úÖ Context-aware validation (valid for "posts")
    startDate: new Date("2024-01-01"), // ‚úÖ Date object validation
    limit: 20                     // ‚úÖ Number type enforcement
  }
})

// ‚ùå TypeScript prevents invalid combinations at compile time
const invalid = await sdk.statistics.getStats({
  statType: "posts",
  params: {
    by: "revenue"                 // ‚ùå Not valid for "posts" statType
  }
})
```

### Response Type Safety

All API responses are **fully typed** with IntelliSense support:

```typescript
const result = await sdk.messages.getChatsList({ 
  authentication: { connectionId } 
})

if (result.data) {
  // ‚úÖ TypeScript knows the exact structure
  result.data.list.forEach(chat => {
    console.log(chat.user.username)    // ‚úÖ IntelliSense available
    console.log(chat.lastMessage.text) // ‚úÖ Nested properties typed
    console.log(chat.unreadCount)      // ‚úÖ Number types enforced
  })
}
```

## Authentication Architecture

### Flexible Authentication Options

The SDK supports **multiple authentication modes** with automatic validation:

```typescript
// Connection-based authentication (recommended)
const connectionAuth = {
  authentication: { connectionId: "conn_123" }
}

// Session-based authentication (for direct access)
const sessionAuth = {
  authentication: { 
    session: {
      "user-id": "123456789",
      "user-agent": "...",
      "cookie": "..."
    }
  }
}

// The SDK automatically validates which auth method is required
// ‚úÖ Some operations require sessions, others accept connections

  authentication: { session: userData }, // ‚úÖ Session required for user actions
  userId: 123456789
})
```

## Error Handling Architecture

### Structured Error Types

All errors follow a **consistent structure** with type safety:

```typescript
interface ErrorResponse<T = any> {
  error: {
    statusCode: number          // ‚úÖ HTTP status
    type: ErrorType            // ‚úÖ Categorized error types
    message: string            // ‚úÖ Human-readable message
    retry: boolean             // ‚úÖ Whether operation can be retried
    details?: T                // ‚úÖ Context-specific error data
  }
}

// Usage with type safety
const result = await sdk.user.getUserInfo(params)

if (result.error) {
  switch (result.error.type) {
    case "unauthorized":
      // Handle session expiry
      break
    case "forbidden":
      // Handle permission issues
      break
    case "bad_request":
      // result.error.details is typed based on the operation
      console.log("Validation errors:", result.error.details)
      break
  }
}
```

## Request Flow Architecture

### Unified Request Pipeline

All SDK operations follow the **same request flow**:

```mermaid
graph LR
    A[SDK Method Call] --> B[Parameter Validation]
    B --> C[Authentication Check]
    C --> D[Request Signing]
    D --> E[API Request]
    E --> F[Response Validation]
    F --> G[Type-Safe Response]
    
    B -->|‚ùå Invalid| H[Compile Error]
    C -->|‚ùå Missing Auth| I[Runtime Error]
    E -->|‚ùå HTTP Error| J[Structured Error]
```

### Automatic Request Enhancement

The SDK **automatically enhances** every request:

```typescript
// What you write
const result = await sdk.posts.createPost({
  authentication: { connectionId: "conn_123" },
  params: {
    text: "Hello world!",
    price: 5.00
  }
})

// What the SDK does internally:
// 1. ‚úÖ Validates all parameters (TypeScript + runtime)
// 2. ‚úÖ Adds required headers (user-agent, content-type, etc.)
// 3. ‚úÖ Signs request with dynamic rules (if needed)
// 4. ‚úÖ Handles authentication context
// 5. ‚úÖ Manages rate limiting
// 6. ‚úÖ Processes response and provides typed data
// 7. ‚úÖ Converts errors to structured format
```

## Performance Architecture

### Optimized for Scale

The SDK is designed for **high-performance applications**:

```typescript
// ‚úÖ Efficient batch operations
const [messages, posts, stats] = await Promise.all([
  sdk.messages.getChatsList({ authentication: { connectionId } }),
  sdk.posts.getMyPosts({ authentication: { connectionId } }),
  sdk.statistics.getStats({ 
    authentication: { connectionId },
    statType: "posts" 
  })
])

// ‚úÖ Built-in pagination support
async function getAllChats(connectionId: string) {
  let allChats = []
  let offset = 0
  const limit = 50  // SDK validates pagination parameters
  
  while (true) {
    const result = await sdk.messages.getChatsList({
      authentication: { connectionId },
      params: { limit, offset }
    })
    
    if (!result.data?.list.length) break
    
    allChats.push(...result.data.list)
    offset += limit
  }
  
  return allChats
}
```
