## Earnings Chart

Get time-series earnings data for a connection. Use `startDate` and `endDate` to specify a date range, and `by` to break down by earning type.

```python
from onlyfans_sdk import OFAuthClient, earnings

client = OFAuthClient(api_key="your-api-key", connection_id="conn_xxx")

chart = earnings.list_charts(
    client,
    start_date="2024-01-01",
    end_date="2024-01-31",
    by="total",
    with_total=True,
)
```

**`by` parameter options:** `total`, `messages`, `tips`, `stream`, `post`, `subscribes`, `tips_profile`, `tips_post`, `tips_chat`, `tips_stream`, `tips_story`, `ref`

## Transactions

List individual earnings transactions, optionally filtered by type.

```python
# List transactions
transactions = earnings.list_transactions(
    client,
    start_date="2024-01-01",
    type="tips",
    tips_source="chat",
)

# Auto-paginate through all transactions
for tx in earnings.iter_transactions(client, type="tips"):
    print(tx)
```

## Chargebacks

List chargeback records with optional date range filtering.

```python
chargebacks = earnings.list_chargebacks(
    client,
    start_date="2024-01-01",
    end_date="2024-01-31",
    limit=20,
)

# Auto-paginate
for cb in earnings.iter_chargebacks(client, start_date="2024-01-01"):
    print(cb)
```

## Analytics

Content performance analytics are available across multiple categories. Each category offers a **chart** endpoint (time-series data) and a **top** endpoint (ranked content).

<Info>
  All analytics endpoints require the `analytics:read` permission.
</Info>

### Posts Analytics

```python
from onlyfans_sdk import analytics

# Posts chart
posts_chart = analytics.list_posts_charts(
    client,
    start_date="2024-01-01",
    end_date="2024-01-31",
    by="views",
    with_total=True,
)

# Top performing posts
top_posts = analytics.list_posts_tops(
    client,
    by="purchases",
    start_date="2024-01-01",
    end_date="2024-01-31",
    limit=10,
)

# Stats for a specific post
post_stats = analytics.get_posts(client, post_id="123456")
```

### Mass Messages Analytics

```python
# Mass messages chart
mm_chart = analytics.list_mass_messages_charts(
    client,
    start_date="2024-01-01",
    end_date="2024-01-31",
    with_total=True,
)

# List purchased mass messages
purchased = analytics.list_mass_messages_purchaseds(client, limit=20)

# List buyers for a specific mass message
buyers = analytics.list_mass_messages_buyers(client, mass_message_id="456", limit=20)

# Auto-paginate through all buyers
for buyer in analytics.iter_mass_messages_buyers(client, mass_message_id="456"):
    print(buyer)
```

### Additional Analytics

The SDK also provides analytics for **trials**, **campaigns**, and **visitor countries**:

```python
# Trials analytics
trials_chart = analytics.list_trials_charts(client, start_date="2024-01-01", end_date="2024-01-31")
top_trials = analytics.list_trials_tops(client, start_date="2024-01-01", end_date="2024-01-31", limit=10)

# Campaigns analytics
campaigns_chart = analytics.list_campaigns_charts(client, start_date="2024-01-01", end_date="2024-01-31")
top_campaigns = analytics.list_campaigns_tops(client, start_date="2024-01-01", end_date="2024-01-31", limit=10)

# Visitor countries analytics
countries_chart = analytics.list_visitor_countries_charts(client, start_date="2024-01-01", end_date="2024-01-31", by="total")
top_countries = analytics.list_visitor_countries_tops(client, start_date="2024-01-01", end_date="2024-01-31", by="total")
```

### Analytics Namespace Reference

| Category | Python |
|---|---|
| Posts chart | `analytics.list_posts_charts()` |
| Posts top | `analytics.list_posts_tops()` |
| Post stats | `analytics.get_posts()` |
| Stories chart | `analytics.list_stories_charts()` |
| Streams chart | `analytics.list_streams_charts()` |
| Mass messages | `analytics.list_mass_messages_charts()` |
| Promotions | `analytics.list_promotions_charts()` |

<Warning>
  When querying large date ranges, use reasonable `limit` values and pagination to avoid timeouts. The `startDate` and `endDate` parameters accept `YYYY-MM-DD` format strings.
</Warning>
